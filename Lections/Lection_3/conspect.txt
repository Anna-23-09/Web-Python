--- PAGE OBJECT ---

- это паттерн автоматизации работы с веб-страницами, который фактически
является стандартом в автоматизации тестирования веб-приложений.
Его основная идея состоит в том, чтобы разделить логику тестов от реализации

Преимущества:
- переиспользование кода
- удобство понимания
- шаблонизация для понятного чтения кода другими разработчиками тестов


Суть PageObject:
- отдельный файла для базовых операций (открытие браузера, инициализация тестов, и т д )
- отдельный файл с описанием методов работы с отдельной страницей

Схема паттерна:
1) Base Page       ->      2) Page Object         -> 3) Tests

1) базовая логика работы веб-страницы (инициализация Selenium WebDriver, открытие веб-страницы)
2) класс (файл с классами) описывает страницу: локаторы и методы, специфичные для работы на этой странице
3) сами тесты
4) и файл conftest.py с фикстурами

Области действия фикстур:
- function
- class
- module
- package
- session

ЛОГИРОВАНИЕ

Библиотека logging

Параметры basicConfig():
- level - уровень логирования
- filename - место, куда мы направляем логи
- format - вид, в котором мы сохраняем результат

Логи в pytest
Параметры:
- log_file_format - структура строк лога
- log_file_date_format - структура метки времени
- log_file - путь к файлу лога
- log_file_level - уровень логирования
