API - это интерфейс, позволяющий 2 независимым
компонентам ПО обмениваться информацией.
Не у всех веб-приложений имеется доступный для
тестирования API, но там, где он есть, его обязательно
нужно протестировать

UI-тестирование - этап тестирования ПО, проверяющий графический интерфейс

Data Driven Testing (DDT) - подход к созданию/архитектуре автоматизированных
тестов (юнит, интеграционных, чаще к backend-тестированию), при
котором тест умеет принимать набор входных параметров и эталонный результат,
с которым он должен сравнить результат, полученный в
ходе прогонки входных параметров.


--- Библиотека requests ---

pip3 install requests

Пример запроса:
response = requests.get('https://api.github.com')

Код ответа получаем с помощью команды:
response.status_code
Если получаем код в промежутке 200-400, выведется True, в противном случае - False

- response.content - получить все содержимое компонента в байтах (неудобно)
- response.text - получить текст ответа (более-менее)
- response.json() - самый удобный

- параметр headers={'Accept': 'application/vnd.github.v3.text-match+json'
- параметр params={'q', 'requests+language:python'}
